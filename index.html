🚀 Getting started: setup steps

1
Create your Google Sheet

Sign in as scarboroughsdamusicdepartment@gmail.com, go to sheets.google.com, create a new spreadsheet. Add 4 tabs named exactly: Members, Groups, Events, Admins. Column headers are listed at the top of AppsScript_Code.gs in this repo.

2
Add yourself to the Admins tab

Put Email in cell A1 of the Admins tab, then list every Gmail address that should have admin access in column A below it. Only emails on this list can sign in to this admin panel.

3
Deploy the Apps Script

In the Sheet, go to Extensions → Apps Script. Delete the starter code and paste in the full contents of AppsScript_Code.gs from this repo. Click Deploy → New deployment → type "Web app" → Execute as "Me" → Who has access "Anyone" → Deploy. Copy the Web App URL it gives you.

4
Connect the website to your Sheet

In index.html, find const DATA_URL near the top of the script and paste your Web App URL in. Commit the change to GitHub.

5
Set up Google sign-in

Go to console.cloud.google.com, create a project, create an OAuth Client ID (type: Web application), and add your GitHub Pages URL as an authorized JavaScript origin. Paste the Client ID into const GOOGLE_CLIENT_ID in index.html.

6
Turn on daily reminder emails

In the Apps Script editor, select setupDailyTrigger from the function dropdown, click Run once. This schedules a daily 8am check that emails anyone whose chosen reminder window matches an upcoming event.

📁 Where things live

index.html — the website, in your GitHub repo.
AppsScript_Code.gs — the backend logic, lives inside your Google Sheet's Apps Script editor.
The Google Sheet — your real database. Members, Groups, Events, and the Admins list all live here as plain rows you can edit directly if needed.
